# Workflows

## Wöchentliches Review (Sonntags)

Jeden Sonntag: `/weekly-synthesis` ausführen

Ablauf:
1. `git pull` – neueste Änderungen holen
2. Inbox leeren und verarbeiten
3. Aktive Projekte durchgehen (Hochzeit, Fett to Fit)
4. Bereiche aktualisieren (Gesundheit, Finanzen, Karriere)
5. Archiv prüfen
6. Commit & Push

## Inbox-Management

- Inbox ist temporär, kein dauerhafter Speicherort
- Ziel: < 20 Items jederzeit
- Wöchentlich verarbeiten: Erfassen → Verarbeiten → Organisieren

## Git-Workflow

```bash
# Session starten
git pull

# Nach Änderungen
git add .
git commit -m "Kurze Beschreibung"
git push
```
