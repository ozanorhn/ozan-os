# Intermediate Packets – Wiederverwendbare Bausteine

> [!quote] Kern
> Statt jedes Mal von null anzufangen, baust du eine Bibliothek von Modulen, die du in neue Projekte einbauen kannst.

## Worum geht's

Ein **Intermediate Packet (IP)** ist ein kleines, in sich abgeschlossenes Ergebnis: eine strukturierte Notiz, ein Diagramm, eine Liste, ein Skript, eine Checkliste. Es ist groß genug, um nützlich zu sein, klein genug, um in mehrere zukünftige Projekte zu passen.

## Beispiele aus Ozans Vault

- Die `/forte`-, `/goggins`-, `/sharma`-, `/clear`-Skills → IPs für Reviews
- Buchnotizen → IPs für künftige Reflexion, Coaching, Schreibarbeit
- Cookie Jar – Meine Siege → IP für mentale Reserven (Goggins-Werkzeug)
- Vermögensübersicht → IP für Finanz-Reviews
- Diese atomaren Themen-Notizen → IPs für künftige Schreibarbeit, Vorträge, Antworten

## Vorteil

- Jedes neue Projekt startet **nicht** bei null
- Output wird modularer, schneller, konsistenter
- Über Zeit wird die IP-Bibliothek selbst zum Wettbewerbsvorteil

## Praxis

- Beim Erstellen: frage dich *„In welchem zukünftigen Projekt könnte ich das wiederverwenden?"*
- Wenn die Antwort „in keinem" lautet → vielleicht braucht es das Packet nicht.
- Benenne IPs so, dass sie auch ohne Kontext verständlich sind.
