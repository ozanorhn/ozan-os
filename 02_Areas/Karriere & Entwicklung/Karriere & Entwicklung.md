## 2026 – Ein Thema pro Quartal (aus [[06_Metadata/Plans/2026-04-19-jahresplan-2026|Jahresplan 2026]])

| Quartal | Fokus-Thema | Output-Ziel |
|---------|-------------|-------------|
| **Q2 2026** (Apr–Jun) | Claude API Advanced (Prompt Caching, Tool Use, Agents) | 1 Output/Monat (LinkedIn-Post oder Demo) |
| Q3 2026 (Jul–Sep) | n8n Skalierung (Workflows, Self-Hosting, Performance) | Meetup-Kurzvortrag oder Blog |
| Q4 2026 (Okt–Dez) | Supabase + Vector DB (RAG-Pattern) | Public Demo-Repo |

**Prinzipien:**
- **Sichtbarkeit = Output, nicht Networking-Events.** Stack (n8n + Claude API + Supabase) ist Nische, aber gefragt.
- **Lernslot fix:** 1h/Woche Deep Work (Sa oder So) – nicht "wenn Zeit".
- **Ein Thema pro Quartal, nicht drei parallel.**
